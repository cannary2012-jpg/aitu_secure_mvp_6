# Система управления академической успеваемостью студентов AITU (MVP)

Данный репозиторий содержит финальную версию минимально жизнеспособного продукта (MVP) — веб-систему контроля и учета оценок для Astana IT University. Проект разработан и протестирован в соответствии с принципами жизненного цикла безопасной разработки (Secure SDLC) и стандартами OWASP Top 10.

## 🚀 Используемые технологии
* **Backend:** Python 3.12, FastAPI (Uvicorn ASGI-сервер)
* **ORM & СУБД:** SQLAlchemy, SQLite (база данных изолирована внутри контейнера)
* **Frontend:** HTML5, CSS3, JavaScript (взаимодействие через Fetch API)
* **Изоляция и контейнеризация:** Docker, Docker Compose
* **Безопасность (Зависимости):** passlib (хеширование bcrypt), PyJWT (управление сессиями)

---

## 🔒 Реализованные механизмы безопасности
В ходе проектирования и устранения архитектурных дефектов были внедрены следующие защитные меры:
1. **Аутентификация и сессии:** Пароли пользователей не хранятся в открытом виде. Применяется криптографическое хеширование на базе алгоритма `bcrypt`. Авторизация пользователей реализована через криптографически подписанные токены `JWT (JSON Web Tokens)`.
2. **Разграничение прав доступа (RBAC):** На уровне API реализован строгий контроль ролей (`student`, `teacher`, `dean`). Доступ к изменению данных и массовому просмотру ограничен (преподаватели видят всю выборку, студенты — только свои оценки).
3. **Безопасность конфигурации:** Все конфиденциальные параметры (секретные ключи подписи) вынесены из исходного кода в переменные окружения (`.env` файл) и исключены из репозитория.
4. **Защита от DoS/Буфера:** Внедрен HTTP-middleware, принудительно ограничивающий максимальный размер любого входящего запроса до 512 КБ для защиты оперативной памяти контейнера.
5. **Валидация данных:** Входные данные строго типизированы с помощью схем `Pydantic v2`. Текстовые поля (например, названия предметов) проходят экранирование через `html.escape()` для защиты от XSS и инъекций.
